# Docs Orchestration — manifest, sequence, and the generation contract

Accessed via [`README.md`](README.md). If you arrived here directly, read that first.

The docs orchestrator is a thin composer. It runs the documentation pipeline for a racecar repo by calling the homes that already own each capability, and it owns only three things nothing else does: the required-docs manifest (below), the [content-blindness contract](CONTENT_BLINDNESS.md), and the sequence and generation contract on this page. Pair with [`../doc-coherence/PROTOCOL.md`](../doc-coherence/PROTOCOL.md) for the doc update protocol it honors and [`../llm-summary/SPEC.md`](../llm-summary/SPEC.md) for the brief it regenerates.

## Required-docs manifest

The canonical set of docs a racecar repo must own, split into two tiers by owner so no rule is stated twice (one home per rule):

**Repo-root tier — owned here, mechanized by [`scripts/check_required_docs.py`](scripts/check_required_docs.py):**

1. **`README.md`** — the human storefront and doc-graph root. Non-empty, opening with a YAML frontmatter block that carries a `pnode` key (the doc-graph edge, [`../doc-coherence/DOC_GRAPH.md`](../doc-coherence/DOC_GRAPH.md)) and, once declared, the `content_blind` policy ([CONTENT_BLINDNESS.md](CONTENT_BLINDNESS.md)).
2. **`CLAUDE.md`** — the agent baseline / resolver. Non-empty, at least one `##` heading. Its recommended four-section shape is doc-coherence's, not restated here ([`../doc-coherence/PROTOCOL.md`](../doc-coherence/PROTOCOL.md), "CLAUDE.md shape").
3. **`docs/summary/$REPO.md`** — the racecar-llm-summary brief. Required unless a repo opts out with `[tool.racecar.required-docs] brief = false`.

**Subsystem tier — owned by doc-coherence, NOT re-implemented here:** every major subsystem in an import-linter layer owns `README.md` + `CLAUDE.md`, enforced by [`../doc-coherence/scripts/check_subsystem_docs.py`](../doc-coherence/scripts/check_subsystem_docs.py). The orchestrator runs that checker; it does not restate its manifest. This is the reconciliation the manifest is deliberately drawn around: `check_required_docs.py` covers the repo root, `check_subsystem_docs.py` covers the subsystems, and the two sets do not overlap.

**Not required, on purpose.** `ARCHITECTURE.md` is a legitimate *pointer target* from `CLAUDE.md`'s architecture section, not a mandated file. `check_subsystem_docs.py` mandates README + CLAUDE and nothing more, so promoting `ARCHITECTURE.md` to required would invent a set racecar does not already hold. Where a repo has one, cite it; the orchestrator neither requires nor generates it.

## The sequence

Four stages, in dependency order. Each later stage assumes the earlier one's output, so the order is not cosmetic. The generative stages (1 and 2) need judgment and are the agent's to drive; the gate stages (3 and 4) are deterministic and are what [`scripts/docs_orchestrate.py`](scripts/docs_orchestrate.py) runs. Ownership is not delegable ([`../shared/OWNERSHIP.md`](../shared/OWNERSHIP.md)): the model authors the narrative, the script confirms it.

1. **Generate the missing required docs.** Run `check_required_docs.py`. For each *missing* repo-root doc, stub the skeleton (frontmatter + the section headings its owner-doc prescribes) and leave the narrative to the owner. An *existing* doc is never regenerated wholesale here (see the generation contract). A missing brief is generated by driving [`/racecar-llm-summary`](../llm-summary/SKILL.md), not by hand.
2. **Regenerate the machine-derivable spine.** The spine is everything re-derived from the module tree, so it cannot drift and is safe to rewrite:
   - the brief's YAML frontmatter (entities / relationships / external surface), regenerated per [`../llm-summary/SPEC.md`](../llm-summary/SPEC.md);
   - the CLI surface docs — the `docs/cli/**/README.md` tree (one page per CLI node) plus the repo README's `## CLI` block — regenerated by [`../arch-coherence/scripts/gen_cli_docs.py`](../arch-coherence/scripts/gen_cli_docs.py), which projects the audited `python -m <pkg>…` tree (it reuses `check_cli_commands.py`'s walk, inventing no structure of its own);
   - the REST / MCP surface docs, regenerated from the Interface Manifest by [`../arch-coherence/scripts/scaffold_surfaces_docs.py`](../arch-coherence/scripts/scaffold_surfaces_docs.py) (OpenAPI, the consolidated `ENDPOINTS.md`, the GET sitemap);
   - When `content_blind: true`, this stage emits structure and placeholders only, never real names or figures (CONTENT_BLINDNESS.md, "Content-blind-aware generation").
3. **Apply the content-blindness gate.** Run [`scripts/check_content_blind.py`](scripts/check_content_blind.py). It is a no-op unless the repo opted in; when it fires it fails the pipeline, so a leaked figure in generated or hand-authored prose cannot ship.
4. **Close with the coherence and link gate.** Run `check_docs.py`, `check_doc_graph.py`, `check_subsystem_docs.py`, and `check_file_placement.py`. Broken links, a cyclic doc graph, an undocumented subsystem, or an orphan doc fails the pipeline.

`docs_orchestrate.py` runs stages 1 (the report half), 3, and 4 as composed gates and returns a single exit code. It resolves each checker whether it lives under racecar's lens dirs or is synced flat into an adopter's `scripts/`, so the same entry point works in both.

## The generation contract: no clobber, but repair

The single most important behavioral guarantee. Regeneration is idempotent and re-runnable, and it PRESERVES all hand-authored narrative. It honors the doc-coherence update protocol ([`../doc-coherence/PROTOCOL.md`](../doc-coherence/PROTOCOL.md), "Update protocol"): PRESERVE the human "why", ADD what is missing, UPDATE what has drifted, never blindly DELETE.

No-clobber is not no-repair. Leaving a drifted machine spine untouched inside an existing doc is a failure, not a courtesy: the spine is re-derived from code and must be brought current every run. The rule is narrow and exact:

- **Fully-generated docs** (OpenAPI, `ENDPOINTS.md`, the sitemap) carry a "Generated by …" header and no hand narrative by construction. They are rewritten wholesale each run. Safe.
- **Mixed docs** (`README.md`, `CLAUDE.md`, the brief) carry hand-authored narrative around a machine spine. The spine lives inside sentinel-delimited regions:

  ```
  <!-- racecar:generated:<spine> start -->
  ...regenerated content...
  <!-- racecar:generated:<spine> end -->
  ```

  Regeneration replaces ONLY the text between a region's sentinels (UPDATE / repair), ADDS a region where a required spine block is absent, and PRESERVES every character outside every region. The brief is the one exception whose sub-generator owns its own preserve-body lifecycle: [`../llm-summary/SPEC.md`](../llm-summary/SPEC.md) re-derives the frontmatter and asks before overwriting the body, so the orchestrator defers to it rather than wrapping it in sentinels.
- **An existing mixed doc with no sentinels** is never silently overwritten. The orchestrator adds the spine region and REVIEW-flags the doc for the owner; it does not guess where the machine content belongs.

Because the spine is re-derived and the narrative is preserved byte-for-byte, running the pipeline twice with no source change is a no-op ([`../shared/PRINCIPLES.md`](../shared/PRINCIPLES.md), P-05).

## Invocation

```
/racecar-docs
/racecar-docs generate the missing docs only, do not regenerate the spine
/racecar-docs regenerate the CLI/REST/MCP surface docs and the brief frontmatter
/racecar-docs make this repo content-blind
```

Default: discover the current repo, run the deterministic backbone (`docs_orchestrate.py`) to report state, then drive the generative stages the report calls for, and re-run the backbone to confirm green. If the README carries no `content_blind` policy, ask once whether the repo is published content-blind and write the answer to frontmatter (CONTENT_BLINDNESS.md); thereafter read it, never re-ask.
